# VERIFACE
> **Verify + Face** — 얼굴로 진실을 검증하다

AI 기반 미스터리 추리 게임. 플레이어가 탐정이 되어 용의자를 심문하고 범인을 밝혀낸다.
웹캠으로 플레이어의 표정을 실시간 감지하여 게임 진행에 반영하는 것이 핵심 차별점이다.

---

## 기술 스택

| 분류 | 기술 |
|------|------|
| 프론트엔드 | React + Vite |
| 백엔드 | FastAPI (Python) |
| AI 스토리/대사 | Gemini 2.5 Flash API |
| 표정 인식 | FER + OpenCV (실시간 웹캠) |
| DB | PostgreSQL + SQLAlchemy |

## 전체 게임 플로우

```mermaid
flowchart TD
    A([게임 시작]) --> B[시네마틱 오프닝\n사건 나레이션 연출]
    B --> C[Gemini API\n사건 자동 생성]
    C --> D[페이즈 1\n현장 조사]
    D --> E{단서 5개\n수집 완료?}
    E -- 아니오 --> D
    E -- 예 --> F[페이즈 2\n용의자 심문]
    F --> G[페이즈 3\n디덕션 보드]
    G --> H[페이즈 4\n최후 심문]
    H --> I{범인\n지목 정확?}
    I -- 성공 --> J([성공 엔딩\n자백 + 사건 해결])
    I -- 실패 --> K([실패 엔딩\n놓친 단서 회고])
    J --> L[플레이 스타일 리포트]
    K --> L
```

## 페이즈별 상세

### 페이즈 1 — 현장 조사

```mermaid
flowchart LR
    A[범죄 현장 입장] --> B[단서 카드 클릭]
    B --> C{단서 연관 용의자\n힌트 표시}
    C --> D[단서 수집 완료\n5개]
    D --> E[심문 페이즈 진입]
```

- 클릭으로 단서 카드 수집
- 각 단서에 용의자 연관 힌트 희미하게 표시 → 심문 동기 부여
- 단서 발견 시 강조 애니메이션

---

### 페이즈 2 — 용의자 심문

```mermaid
flowchart TD
    A[용의자 선택] --> B[질문 톤 선택\n압박 / 공감 / 회유]
    B --> C[자유 타이핑 질문]
    C --> D{웹캠 표정 감지}
    D -- neutral --> E[압박 유지\n긴장도 상승]
    D -- happy --> F[경계 풀림\n추가 단서 획득 기회]
    D -- fear --> G[용의자 우위\n단서 봉인 위험]
    D -- surprise --> H[용의자 진술 변경]
    E & F & G & H --> I[Gemini API\n표정+톤 반영 대사 생성]
    I --> J{거짓 발언\n감지?}
    J -- 예 --> K[텍스트 떨림 효과\n시각 신호]
    J -- 아니오 --> L[일반 대사 출력]
    K & L --> M{용의자\n역심문?}
    M -- 예 --> N[플레이어 선택지 3개\n대답 선택]
    M -- 아니오 --> O{모든 용의자\n심문 완료?}
    N --> O
    O -- 아니오 --> A
    O -- 예 --> P[디덕션 보드]
```

**표정별 전략 효과**

| 감지 표정 | 전략 의미 | 게임 효과 |
|-----------|-----------|-----------|
| 😐 neutral | 포커페이스 유지 | 압박 지속, 긴장도 서서히 상승 |
| 😄 happy | 의도적 친화 | 경계 풀려 추가 정보 획득 가능 |
| 😮 surprise | 감정 노출 | 용의자가 눈치채고 진술 변경 |
| 😨 fear | 약점 노출 | 용의자 우위, 단서 봉인 위험 |

### 페이즈 3 — 디덕션 보드

```mermaid
flowchart TD
    A[수집한 단서 카드 배치] --> B[심문 중 추출된\n진술 카드 배치]
    B --> C[드래그로 연결선 긋기\n단서 ↔ 진술]
    C --> D{연결 판정}
    D -- 올바른 연결 --> E[초록 확인\n연결 확정]
    D -- 틀린 연결 --> F[빨간 경고\n패널티 발생]
    E --> G{필요 연결 수\n달성?}
    F --> G
    G -- 아니오 --> C
    G -- 예 --> H[범인 지목 가능\n최후 심문 버튼 활성화]
```

- 단서 카드와 진술 카드를 직접 연결하는 셜록 홈즈식 추론 보드
- 같은 단서도 두 가지 해석 가능 → 방향 선택이 이후 전개에 영향
- 단서를 언제 꺼낼지 타이밍 전략 적용

---

### 페이즈 4 — 최후 심문

```mermaid
flowchart TD
    A[범인 지목 선언] --> B[범인 강력 반박]
    B --> C{웹캠 표정 상태}
    C -- fear/surprise --> D[범인 역공\n추가 반박 강화]
    C -- neutral/happy --> E[결정타 증거 제시\n단서 선택]
    D --> F{플레이어\n버텨냄?}
    F -- 예 --> E
    F -- 아니오 --> G([실패 엔딩])
    E --> H{증거 정확?}
    H -- 예 --> I([성공 엔딩\n자백])
    H -- 아니오 --> G
```

- 표정 감지가 가장 크게 작용하는 구간
- 최후 순간에 얼굴이 흔들리면 범인이 역공
- 수집한 단서 중 "결정타"를 직접 선택해서 제시

## 엔딩 & 리포트

```mermaid
flowchart LR
    A{엔딩 분기} -- 성공 --> B[자백 씬\n사건 해결 나레이션]
    A -- 실패 --> C[증거 불충분\n놓친 단서 회고 씬]
    B --> D[플레이 스타일 리포트]
    C --> D
    D --> E[압박형 vs 공감형 비율]
    D --> F[표정 통계\nneutral 유지 시간 등]
    D --> G[획득/봉인된 단서 수]
    D --> H[심문 스타일 총평]
```

## VERIFACE 고유 차별점

> 다른 추리 게임은 **게임 속 정보**만 변수다.
> VERIFACE는 **플레이어 본인**이 변수다.

- 웹캠이 탐정인 "나"를 보고 있다
- 내 표정, 질문 톤, 반응 속도가 모두 게임에 영향
- 같은 사건도 플레이어마다 완전히 다른 경험
- 오답도 허용 — 실패가 좌절이 아닌 "다시 하고 싶다"로 연결
